Pensamento computacional

  • surgiu na década de 50 por Babbage
  • é a capacidade de sistematizar representar e resolver problemas
  • é uma habilidade fundamental para todos
  • processos de pensamento envolvidos na formulação de um problema e de soluções eficazes de forma que uma máquina ou pessoa possa realizar
  • é uma similaridade de pensamento comum para máquinas e pessoas
  • é aplicado para descrever, explicar e modelar o universo e seus processos complexos

Computação plugada: utilização de computadores na resolução de problemas

Computação desplugada: são resoluções de problemas por atividades manuais (jogos, desenhos etc) para resolução de problemas

Pilares do Pensamento Computacional

decomposição do problema -> reconhecimento de padrões -> criação de abstrações -> criação de algoritmos

  • Decomposição: dividir o problema em partes menores para facilitar a solução
  • Reconhecer padrões: quanto mais padrões forem encontrados mais fácil se torna a resolução do problema. Muito utilizado por seres humanos
  • Abstração: filtragem e classificação dos dados. Separar o que é essencial do problema. Dispensar o irrelevante
  • Algoritmo:sequencia de passos finitos para chegar à solução

Paradigmas de programação

meio de classificar as linguagens de programação baseado em suas funcionalidades

são classificadas dos seguintes tipos de paradigma

  • Imperativo: especifica os passos a serem seguidos. Subdivide-se em:
    1. procedurais: trabalha com instruções ou procedimentos (passos) (Fortran, Cobol, Pascal e Basic)
    2. Orientada a Objeto: define objetos e as relações entre eles (mundo real) (Java, C#, Python, Ruby)
  • Declarativo: trabalha na relação ou função entre os dados. Subdivide-se em:
    • funcional: funções matemáticas para realizar computações. Evita-se dados que são mutáveis (Prolog, Resquion)
  • Estrutura de Tipos
    1. fracamente tipada: não define o tipo das variáveis (PHP)
    2. fortemente tipada: os tipos das variáveis são imutáveis e pré-definidos (Java)
    3. dinamicamente tipada: modernas. Define o tipo da variável em tempo de execução (Python, Ruby)
    4. estaticamente tipada: define o tipo da variável no momento da compilação (Java, C)
  • Grau de Abstração: escala para a linguagem (próximo a máquina ou do ser humano)
    • baixo nível: Assembly
    • médio nível: C++, Java e Python
    • alto nível
  • Geração: primeira, segunda, terceira, quarta e quinta

Trade-off

Confiabilidade e custo de execução. Exemplo: em Java, um vetor deve ser completamente validado, o que aumenta o custo computacional da execução. Java é confiável, porém consome muitos recursos computacionais.

Ponteiros: em C++ pode-se alterar facilmente os ponteiros porém a confiabiliade decai.

  • Simplicidade: afeta a legibilidade (menos instruções)

Pré-processadores

programa que processa outro programa imediatamente antes dele ser compilado.

Métodos de implementação da linguagem de programação

  • Compilação: traduzidos para a linguagem de máquina (rápido). C, C++ e Rust
  • Interpretação pura: programas são interpretados por outros. Oposto da compilação (mais lento que compilar). Python, Ruby e JavaScript
  • Híbridos: primeiro compila depois interpreta (muito lento). Java (compila em bytecode e interpreta em JVM), C# (compila e interpreta em CLR - Commom Language Runtime) e Golang

Projeto de desenvolvimento de software, algoritmo e implementação

  • Projeto
    • é um esforço temporário (início e fim)
    • a finalidade é um resultado único (criar novo produto ou serviço)
    • tem escopo inicial (descrição parcial) do que trabalho que será feito. Até onde o projeto irá.
    • possui recursos delimitados
    • os custos e recursos (força de trabalho, materiais, verbas, prazos) serão estimados no início do projeto
    • não serve somente para software
  • PMI → organização que dissemina as melhores práticas da gestão de projetos (Project Management Institute)
  • Software
    • produto final do esforço que engenheiros projetaram e construiram. Engloba:
      • programas → produzem o resultado desejado quando em execução
      • documentos → descrevem a operação e uso dos programas
      • estrutura de dados → possibilita a manipulação adequada da informação
      • sistema é algo mais amplo, mas que engloba o software
    • características:
      • maioria feito sob encomenda
      • montado a partir de módulos ou componentes já existentes (não precisa reinventar a roda)
      • é um produto projetado por engenharia
      • não se desgasta mas se deteriora (precisa de atualização e correção)
      • quando o custo da manutenção superar o da criação de novo software então chegou a hora de evoluir.
    • bons atributos:
      • prover funcionalidade e desempenho requerido pelo usuário
      • deve ser confiável
      • fácil de manter
      • fácil de usar
    • modelos de processo de software
      • representação simplificada de um processo de software
      • representam frameworks
      • modelos mais comuns:
        • cascata → definição de requisitos; projeto do sistema; implementação e teste; integragração e teste; operação e manutenção
        • incremental → esboço; desenvolvimento→especificação→validação; versão inicial→versão intermediária→versão final→definição de requisitos; análise de componentes; alterações nos requisitos; projeto do sistema com reúso; desenvolvimento e integração; validação do sistema
        • preditivo
        • adaptativos (ágeis)
        • híbrido
  • Engenharia de software
    • abordagem sistemática, disciplinada e mensurável no processo de construção de um software
    • possui ordem de execução das atividades na construção da solução
    • possui modelos gráficos descritivos do software
    • possui diretrizes com base em boas práticas de projetos
    • principais atividades da engenharia
      • especificação
      • desenvolvimento
      • implementação
      • validação
      • evolução
    • diferença entre engenharia de software e ciência da computação
      • CC → foca a teoria e fundamentos
      • ES → preocupa-se com o desenvolvimento e entrega de softwares úteis
    • precisa-se sempre pensar em fazer software de forma mais rápida e confiável
    • 60% do tempo gasta-se desenvolvendo o software
    • 40% são custos de testes
  • Projeto x Processo
    • projeto de software
      • tempo determinado (início e fim)
      • resultado único
      • trabalho inédito
    • processo de software
      • tempo indeterminado (pode ser as manutenções)
      • resultado padrão
      • trabalho repetitivo
      • tem fases → definição; desenvolvimento; manuteção
      • Def → conjunto de atividades para especificação, projeto, implementação e teste de software
      • modelos → abstração do processo de software. Descrição de um processo a partir de uma perspectiva particular de quem criou
  • Ciclo de vida de projetos
    • início do projeto
    • organização e preparação
    • execução do trabalho
    • fim do projeto
    • são classificados como:
      • Preditivo → o escopo, o prazo e o custo do projeto são determinados nas fases iniciais
      • Interativo → recomendável para projetos que já possuam a definição do que será entregue de maneira clara, porém, com prazo e custos ainda desconhecidos. Cada volta no ciclo de vida chamamos de iteração
      • Incremental → a entrega é realizada por partes e pouco a pouco o produto final vai sendo formado. Cada funcionalidade que é criada um incremento é entregue
      • Adaptativos (ágeis) → utiliza características dos Iterativos e Incrementais somados obtendo, assim, resultados melhores em ambientes com muitas incertezas
      • Híbrido → combinação do ciclo de vida preditivo com o adaptativo. O preditivo espera um ambiente completamente conhecido e o adaptativo um ambiente totalmente incerto, o modelo híbrido busca encontrar um equilíbrio entre os dois
  • Gerência de projetos
    • envolve planejamento, monitoramento e controle de pessoas e processos
    • ocorrem conforme evolui o desenvolvimento do produto (software)
    • tem mais responsabilidade sobre o projeto
    • toma decisão com base no monitoramento
    • tem que ser um agente de mudança
    • estabelece objetivos e busca alcançá-los dentro do prazo e custo estabelecidos
  • Gerente de projetos
    • deve estimular as habilidades do integrantes da equipe (conheça os membros da equipe)
    • deve aperfeiçoar as formas de comunicação
    • deve desenvolver confiança
    • responsável por intermediar a comunicação
    • manter e proteger a coesão da equipe (afastar os problemas)
    • desejável que seja graduado em SI ou CC
    • tenha uma pós-graduação
    • tenha capacidade de liderança
    • ser eficiente na comunicação
    • tenha poder de negociação
    • seja disciplinado e organizado
    • seja persistente
    • tenha visão de longo prazo
    • tenha desenvolvido o poder de tomada de decisão rápida e assertiva
    • saiba como desenvolver os membros da equipe para se tornarem gerentes
    • seja proativo
    • tenha automotivação e otimismo

Padrões importates da web

  • história
    • Vannevar Bush
      • inventor do conceito hipertexto em 1945
      • Memex imita a forma humana de recuperar informação
      • documentos convencionais equivale a escrita linear
    • Ted Nelson
      • criou o termo hipertexto em 1965
      • propôs o Xanadu → sistema de hipertexto que suportaria acesso e armazenamento de hiperdocumentos
      • é o mais próximo do www que usamos hoje
    • Douglas Engelbart
      • inventor do mouse em 1968
      • pai da computação interativa
      • estação de trabalho visual com janelas
    • Tim Berners-Lee
      • apresenta o Mesh em 1989
      • arquitetura cliente-servidor
      • provia uma interface para o cliente
      • permitia acesso ao servidor independentemente do ambiente que estivesse armazenado
      • diversos tipos de documentos através de diversos protocolos
      • um sistema de endereçamento universal
      • apresenta a World Wide Web em 1990
      • o servidor é disponibilizado para o CERN em 1991
      • interfaces cliente/servidor são disponibilizadas publicamente em 1992
      • produz a primeira formalização do HTML
      • 50 servidores instalados no CERN em 1993
      • o Mosaic (browser) surge em set/1993 para os servidores do CERN
      • já eram 500 servidores instalados
    • World Wide Web Consortium (W3C)
      • formado por universidades, centros de pesquisa e indústrias
      • objetiva promover direcionamento estável para a web
      • hoje está localizado no MIT
      • é dirigido por Tim
      • tem representantes na Europa (INRIA na França) e Ásia (Keyo University)
      • tem apoio da DARPA (USA) e da European Comission
    • Rich Internet Application (RIA) 2002
      • denomina aplicações web com características e funcionalidades de software para desktop
      • transfere o processamento das interfaces para os navegadores no lado do cliente
      • mantém o conteúdo no servidor

  • são especificados e desenvolvidos pelo W3C

  • informa as melhores práticas aos desenvolvedores na criação de sites e sistemas

Semana 6

Na década de 1970 e 1980, a Internet era basicamente uma forma de conectar computadores. Já os anos 90 e 2000 marcaram a Internet como a ferramenta de conexão entre as pessoas. Hoje em dia, a ênfase está mudando de forma: conectar tudo (todas as coisas) via Internet.
A internet das coisas(do inglês Internet of Things - IoT) é uma rede de objetos físicos conectados que possuem tecnologia embarcada com sensores com capacidade de coletar dados e transmiti-los. O uso do termo coisa se refere a um objeto do nosso dia a dia colocado em nosso ambiente diário. Por exemplo, podemos encontrar geladeiras inteligentes que monitoram a temperatura do freezer e alertam caso exista uma pane elétrica, cafeteiras que avisam quando o café está pronto, dispositivos que monitoram nossos sinais vitais e os transmitem para um sistema de monitoramento, entre outros.
Nesta semana, vocês aprenderão os fundamentos de IoT, os avanços da tecnologia e conhecerão algumas aplicações e áreas que estão sendo exploradas. Também conhecerão duas tecnologias importantes na implantação de Iot: os protocolos MQTT e REST.

Internet das coisas

  • década de 70 a 80 → era de conectar computadores
  • década de 90 a 2000 → era da conexão entre pessoas (internet)
  • atualmente → era da conexão de tudo (coisas e pessoas) à internet
    • conecta objetos físicos com sensores embarcados para coletar dados
  • baseada em aquisição e análise de dados em tempo real e offline
  • aprendizado de máquina
  • visualização de dados
  • tópicos importantes para big data
  • definições
    • coisa → objeto comum em nosso ambiente
    • dispositivo → sensor, atuador ou tag. Informa dados coletados
    • em geral, um dispositivo pertence a uma coisa (sensor lâmbda, sensor de porta automática etc)
  • arquitetura → dispositivos(hardware pessoais ou vestíveis), gateways(dispositivos de coleta), cloud(armazenamento e comunicação segura) e apps(provém acesso virtual e análise dos dados dos dispositivos). Principais partes:
    • pontos de extremidade → um mais sensores
    • gateways → agregam dados de vários sensores e encaminham para o back-end
    • back-end → armazena e analisa os dados coletados
  • áreas de aplicação:
    • cidades inteligentes → controlar atividades públicas (iluminação)
    • meio ambiente → limpeza do ar e da água
    • agricultura → automatizar irrigação, fertilização do solo, evitar desperdício
    • saúde → melhorar tratamentos
    • alimentos → evitar desperdício
  • IoT x aplicativos móveis e web
    • IoT → são parte de um sistema maior de dispositivos
    • estão sempre ligados
    • são sempre em maior quantidade (rebanho)

    • são heterogêneos (varia recursos de conexão e energia, há de todo tipo)

  • meios de utilização:
    • RFID, 5G, NFC, Bluetooth, Wi-fi ou Xbee

Protocolos MQTT e REST

  • protocolos de comunicação
  • evoluiram de protocolos industriais
  • MQTT (message queue telemetry transport)
    • protocolo do tipo machinte-to-machine
    • é um protocolo leve (na camada de transporte de mensagem)
    • utiliza o modelo de publicação e assinatura
    • é suportado por dois tipos básicos de entidades:
      • broker de mensagem -> grande servidor que recebe todas as mensagens dos clientes
      • clientes -> recebem as mensagens do broker de mensagem
    • possui QoS (qualidade de serviço)
      • QoS 0 -> não confirma a entrega; não armazena a mensagem para retransmissão (perdeu preibói)
      • QoS 1 -> tem confirmação de entrega
      • QoS 2 -> garante a entrega uma vez com confirmação de recebimento
    • é utilizando em uma grande variedade de domínios e ambientes de mercado (automotivo, logística, casas inteligentes, telemetria, saúde, segurança, smarthome, gás, petróleo, ferrovias)
    • REST (representational state tranfer)
      • trabalha com abstração da arquitetura web
      • é um protocolo de comunicação mais recente
      • surgiu com a ideia de simplificar o acesso aos servidores web
      • se baseia no procotolo HTTP
      • permite utilizar vários formatos para representação de dados (JSON, XML, RSS)
      • permite utilizar recursos com identificadores globais
      • permite manipular recursos com componentes da rede
      • a comunicação ocorre através da internet por aparelhos que utilizam o HTTP
    • REST x MQTT
      • REST MQTT
      conexão unidirecional conexão bilateral
      consumo alto de energia consome 20% menos
      sensível à falta de energia conexão persistente
      necessita estabelecer conexão transfere dados até 25 vezes mais rápido
      1k conexões simultâneas 50k conexões simultâneas
    • outros protocolos
      • 6LoWPAN
      • ZigBee
      • AMQP (advanced message queuing protocol)
      • CoAP (contrained application protocol)
    • emuladores para IoT
      • permitem simular o IoT sem o hardware
      • os principais com GUI são (Cooja, OMNet++, Catalia, Sinalgo)
      • são de código aberto e utilizam C, C++ e Java

Data Science - Data Analytics e Big Data

  • atuam com o mesmo objeto: dados
  • voltada ao estudo e análise de dados econômicos, financeiros e sociais
  • podem ser dados estruturados ou não
  • visa extrair conhecimento dos dados, detectar padrões ou insights para tomadas de decisão
  • precisa saber coletar, extrair, classificar, examinar e comparar os dados
  • a principal diferença entre Big Data e Data Science é o volume de dados
  • áreas de atuação
    • e-commerce → recomendar produtos, identificar demandas
    • indústria → prevenir problemas, monitorar sistemas, automação, detectar anomalias
    • financeiro → segmentação de cliente, análise de risco, tomada de decisão
    • saúde → criar drogas, bioinformática, médico virtual, análise de imagens
    • transporte → monitorar veículos, segurança do passageiro, piloto automático
    • bancário → detectar fraudes, análise de risco, seguros
  • carreira em Data Science
    • vantagens → está na moda; muitas vagas; bem remunerada; versátil
    • desvantagens → termo obscuro; domínio impossível; precisa estudar muito; dados arbitrários pode gerar resultados inesperados; privacidade dos dados
